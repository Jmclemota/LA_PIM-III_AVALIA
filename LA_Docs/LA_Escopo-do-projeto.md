# LA_Escopo do Projeto – Avalia+

---

## Escopo do Produto

O presente projeto tem como objetivo o desenvolvimento de uma **plataforma web de avaliação acadêmica**, denominada **Avalia+**, voltada ao apoio de estudantes do ensino superior na preparação para avaliações acadêmicas.

A plataforma permitirá a realização de simulados baseados em um banco de questões organizadas por disciplinas, possibilitando a análise do desempenho dos estudantes e o acompanhamento de indicadores educacionais.

O sistema disponibilizará as seguintes funcionalidades principais:

- Cadastro e autenticação de usuários (Aluno, Professor e Administrador)
- Gerenciamento de disciplinas e banco de questões pelo Administrador
- Gerenciamento de turmas com vinculação de professores e alunos
- Geração automática de simulados com 10, 15 ou 20 questões
- Realização de simulados online pelos estudantes
- Correção automática das avaliações realizadas
- Visualização do histórico de simulados e desempenho individual pelo aluno
- Dashboards de desempenho por turma para professores
- Painel de indicadores administrativos para o Administrador
- Controle de permissões conforme o perfil de usuário
- Visualização das informações do perfil do usuário

A plataforma será desenvolvida como **aplicação web**, utilizando arquitetura **cliente-servidor** e **banco de dados relacional** para armazenamento das informações.

Nesta fase inicial do projeto, a plataforma será disponibilizada com **acesso gratuito aos usuários**, com o objetivo de validar a utilização da ferramenta no contexto acadêmico. Funcionalidades relacionadas a planos pagos ou assinatura institucional poderão ser implementadas em versões futuras.

---

## Escopo Técnico

O sistema será desenvolvido utilizando as seguintes tecnologias e práticas de desenvolvimento:

- **Frontend:** HTML, CSS e JavaScript
- **Backend:** C# — com entrega do **Diagrama de Classes** como representação da arquitetura do sistema nesta fase do projeto
- **Banco de Dados:** SQL Server (banco de dados relacional)
- **Interface:** responsiva, permitindo acesso em diferentes resoluções de tela
- **Código:** organização modular, facilitando manutenção e evolução do sistema
- **Metodologia:** Scrum, para gerenciamento e acompanhamento do desenvolvimento

Essa estrutura técnica busca garantir **manutenibilidade, escalabilidade e organização do software**, conforme boas práticas de engenharia de software.

---

## Fora do Escopo

Para garantir a viabilidade do projeto dentro do prazo estabelecido, algumas funcionalidades **não serão contempladas nesta versão inicial do sistema**.

Estão fora do escopo da versão atual:

- Desenvolvimento de **aplicativo mobile nativo**
- Integração com **sistemas institucionais externos**
- Implementação de **sistema de pagamento integrado**
- Utilização de **inteligência artificial** para geração automática de questões
- Implementação de **dashboards analíticos avançados ou gráficos complexos**
- Elaboração do **Diagrama de Implantação**

Essas funcionalidades poderão ser avaliadas e implementadas em **versões futuras da plataforma**, conforme evolução do projeto e necessidades identificadas durante a utilização do sistema.
