# LA_Sprint Retrospective – Sprint 2

**Projeto:** Avalia+  
**Período da Sprint:** Abril de 2026

---

# Objetivo da Retrospectiva

A Sprint Retrospective tem como objetivo analisar o desempenho da equipe durante a Sprint 2, avaliando os avanços na modelagem do sistema, organização das atividades e evolução da documentação do projeto.

---

# O que funcionou bem

* Desenvolvimento dos diagramas UML do sistema  
* Evolução da modelagem do banco de dados  
* Organização dos fluxos de usuário  
* Melhor alinhamento das responsabilidades entre os integrantes  
* Avanço consistente da documentação técnica do projeto  
* Boa organização das Sprints e Dailys  

---

# Dificuldades encontradas

* Necessidade de revisões constantes na modelagem do sistema  
* Ajustes nos relacionamentos do banco de dados durante o desenvolvimento do MER e DER  
* Dificuldade inicial na padronização da documentação técnica  
* Necessidade de refinamento das regras de negócio do sistema  

---

# Ajustes realizados na equipe

Durante a Sprint 2, a equipe reorganizou parte das responsabilidades para melhorar o fluxo de trabalho e a distribuição das atividades.

As principais mudanças envolveram:

* Maior foco da Scrum Master na organização das Sprints, Dailys, banco de dados e UX/UI  
* Redistribuição das atividades relacionadas à modelagem backend e diagramas  
* Melhor separação entre tarefas de documentação, modelagem e estrutura visual do sistema  

Esses ajustes contribuíram para melhorar a organização interna da equipe e otimizar o andamento do projeto.

---

# Melhorias para a próxima Sprint

Para melhorar o desenvolvimento nas próximas etapas, a equipe definiu as seguintes ações:

* Refinar a modelagem backend em C#  
* Evoluir o diagrama de classes do sistema  
* Melhorar a organização visual das interfaces e protótipos  
* Revisar continuamente os requisitos e regras de negócio  
* Manter a documentação técnica sempre atualizada  
* Melhorar o detalhamento das tarefas dentro das Sprints  

---

# Conclusão

A Sprint 2 foi importante para consolidar a estrutura lógica e organizacional do projeto Avalia+.

A equipe conseguiu avançar significativamente na modelagem do sistema, organização do banco de dados e documentação técnica, estabelecendo uma base mais sólida para as próximas etapas do projeto.
